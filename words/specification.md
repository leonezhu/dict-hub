# specification (规格 / 规范 / 说明书)

## 基础信息
**英文**：specification [ˌspesɪfɪˈkeɪʃn]
**中文**：规格 / 规范 / 说明书 / 明细 / 技术要求
**词性**：noun (名词)
**音标**：/ˌspes.ɪ.fɪˈkeɪ.ʃən/

## 概念分析

### 1. 一词多义 vs 一概念多词
- **英语**：specification → ①规格 ②规范 ③说明书 ④明细 ⑤技术要求
- **汉语**：规格 → specification (产品参数) / 规范 → specification (行业标准) / 说明书 → specification (文档)
- **分析**：典型的**技术领域一词多义**。英语用单一抽象名词涵盖技术文档、参数标准、行业规范等多个维度，汉语则根据具体应用场景使用不同词汇。

### 2. 上下义关系
- **上义词**：document (文档), standard (标准), requirement (要求)
- **下义词**：technical specification (技术规格), product specification (产品规格), design specification (设计规范)
- **汉语表达**：技术规格书 → 产品规格 → 参数明细

### 3. 同义词网络
- **英语同义词**：
  - *文档*：spec sheet, technical document, product manual
  - *标准*：standard, requirement, criterion
  - *参数*：parameter, detail, attribute
- **汉语近义词**：
  - *文档*：说明书, 技术手册, 产品手册
  - *标准*：规范, 标准, 准则
  - *参数*：规格, 参数, 指标
- **角度差异**：英语强调"详细说明"的本质，汉语更注重功能分类（文档/标准/参数）

## 英汉对比

| 特征 | 英语 | 汉语 | 洞察 |
|------|------|------|------|
| **概念细分** | 单一词涵盖文档+标准+参数 | 需要3类词汇分别表达 | 英语重抽象概括，汉语重功能区分 |
| **表达习惯** | 名词化倾向，抽象程度高 | 复合词+具体化，功能导向 | 英语技术文档更简洁，汉语更直观 |
| **翻译对应** | 1词→多词（根据语境选择） | 多词→1词（不同领域用不同词） | 翻译时必须识别具体技术领域 |

## 实际应用

### 场景 1：产品开发
- **英文**："Please review the product **specification** before manufacturing."
- **中文**："请在生产前审阅产品**规格书**。"
- **分析**：此处指详细的产品参数文档，用"规格书"最贴切，不能用"规范"或"说明书"。

### 场景 2：行业标准
- **英文**："This material meets the industry **specification** for fire resistance."
- **中文**："这种材料符合行业**防火规范**。"
- **分析**：此处指行业技术标准，用"规范"最合适，强调合规性。

### 场景 3：技术参数
- **英文**："The **specification** includes dimensions, weight, and material."
- **中文**："**规格**包括尺寸、重量和材料。"
- **分析**：此处指具体参数指标，用"规格"最准确，强调可测量性。

### 场景 4：软件工程
- **英文**："We need to write functional **specifications** for each module."
- **中文**："我们需要为每个模块编写功能**需求说明书**。"
- **分析**：此处指功能需求文档，用"需求说明书"或"规格说明"，强调文档性质。

## 学习建议

### 记忆策略
- **词根记忆法**：speci（特殊）+ fic（做）+ ation（结果）→ "做出特殊要求的东西"
- **场景联想法**：想象工程师拿着spec sheet（规格表）对照产品参数
- **分类记忆法**：记住三个核心场景 → 产品规格、行业规范、技术文档

### 使用注意
- ⚠️ **避免直译错误**：不能说"这个产品的 specification 很好"（模糊），应该明确"产品规格很详细"或"符合规范"
- ⚠️ **注意领域差异**：软件行业常用"需求规格"，制造业常用"产品规格"，建筑业常用"施工规范"
- ⚠️ **缩写使用**：口语中常用"spec"（/spek/），但正式文档必须用全称

### 关联词汇
- **相关词**：specify（动词：详细说明）, specific（形容词：具体的）, spec sheet（规格表）
- **对比词**：standard（标准）, requirement（要求）, regulation（法规）
- **复合词**：technical specification（技术规格）, design specification（设计规范）, product specification（产品规格）

## 深度洞察

### 1. 语言特征
**英语的"技术抽象化"思维**：Specification 体现了英语技术语言的**名词化倾向**——将动词过程（specify）转化为静态概念（specification），便于技术文档的标准化和抽象化表达。这种抽象性使得单一词汇能跨越多个技术领域。

**汉语的"功能具体化"思维**：汉语坚持将抽象的技术概念分解为具体的功能类别——"规格"（参数）、"规范"（标准）、"说明书"（文档），反映了东方文化中对**实用性和可操作性**的重视。

### 2. 概念映射
**多对多映射的复杂性**：
- 英语 `specification` ↔ 汉语 `规格` + `规范` + `说明书` + `明细`
- 这不是简单的翻译问题，而是**技术思维模式**的差异
- 翻译时必须识别：是参数？是标准？还是文档？

**决策树**：
```
上下文是产品参数？ → 用"规格"
上下文是行业标准？ → 用"规范"
上下文是技术文档？ → 用"说明书/规格书"
上下文是详细清单？ → 用"明细"
```

### 3. 学习策略
**掌握 specification 的三步法**：
1. **核心概念**：记住"详细说明的技术要求"这一本质
2. **场景分类**：识别使用场景（产品/标准/文档）
3. **对应选择**：根据场景选择合适的汉语词汇

**避免常见错误**：
- ❌ "这个 specification 很长" → ✅ "这份规格书很详细"
- ❌ "符合 specification" → ✅ "符合规范/规格"
- ❌ "写 specification" → ✅ "编写规格书/制定规范"
